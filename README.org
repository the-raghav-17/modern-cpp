* Learning modern C++
** Modern C++ provides good of both worlds
*** We have high level features just like in modern languages and
*** We have low level control whenever required

** Tips on writing good C++ code
*** Prefer high level features, abstractions and tools over low-level details.
**** Only go low-level when its absolutely necessary (in things like embedded systems)

*** Don't manage memory manually using ~new~ and ~delete~; prefer smart pointers and RAII patterns

*** Use abstractions, containers and tools like ~std::vector~ and ~std::string~ from start, instead of relying on things from vanilla C.
